# Codex 강의 Remotion 내레이션/CTA/타이밍 계획

## 전체 방향

- 톤앤매너: 친근하지만 전문적인 AI 강사. 모든 문장은 존댓말로 작성합니다.
- TTS 기준: 한 문장은 짧게 유지하고, 괄호 설명과 특수기호 읽기를 줄입니다.
- 화면 기준: 강사용 스크립트, 자막, 빨간 박스는 화면에 표시하지 않습니다.
- 애니메이션 기준: 슬라이드 이미지가 이미 CTA 구조를 담고 있으므로, Remotion은 이미지 안의 흐름을 추론해 줌인, 팬, 줌아웃으로 안내합니다.
- 전체 길이: 14장, 약 6분 18초입니다.

## 슬라이드별 계획

### 01. AI에게 묻기에서, AI에게 맡기기로

- 길이: 30초
- CTA 의도: 질문형 AI에서 결과물 제작형 AI로 관점을 전환합니다.
- 화면 전달: 전체 화면에서 Assistant 쪽을 보여준 뒤 Agent 쪽으로 이동하고, 마지막에는 두 개념을 한 화면에 다시 잡습니다.
- Remotion 효과:
  - 0-5초: 전체 맥락, 천천히 줌인
  - 5-12초: Assistant 영역으로 팬
  - 12-22초: Agent 영역으로 팬
  - 22-30초: 전체 비교로 줌아웃
- TTS 스크립트:
  안녕하세요. 오늘은 인공지능을 단순히 질문에 답하는 도구로 쓰는 단계를 넘어, 실제 업무를 맡기는 방법을 배워보겠습니다. 왼쪽은 질문하고 답을 받는 어시스턴트의 장면이고, 오른쪽은 파일과 결과물을 만들어 내는 에이전트의 장면입니다. 오늘의 목표는 명확합니다. 설명을 듣고 끝내는 것이 아니라, 내 업무 하나를 실제 결과 파일로 받아 보는 경험을 만드는 것입니다.

### 02. 작게 요청하고, 파일로 받으면 실습이 쉬워집니다

- 길이: 27초
- CTA 의도: 작은 자료, 명확한 산출물, 결과 검토라는 세 단계만 성공시키게 합니다.
- 화면 전달: 공식 전체를 보여준 뒤 작은 자료, 산출물, 검토 단계 순서로 이동합니다.
- Remotion 효과:
  - 0-4초: 전체 공식에서 부드럽게 줌인
  - 4-10초: 작은 자료 단계로 팬
  - 10-18초: 산출물 지정 단계로 이동
  - 18-27초: 검토와 저장 단계로 팬 후 줌아웃
- TTS 스크립트:
  처음부터 거대한 자동화를 만들려고 하면 누구나 막힙니다. 그래서 오늘은 작게 시작합니다. 자료 하나를 고르고, 원하는 산출물의 형태를 정하고, 결과 파일을 열어 확인합니다. 이 세 단계만 안정적으로 반복해도 Codex 실습은 훨씬 쉬워집니다. 작은 성공을 먼저 만들고, 잘 된 프롬프트를 다음 업무의 출발점으로 저장해 보겠습니다.

### 03. Assistant는 옆에서 돕고, Agent는 대신 수행합니다

- 길이: 27초
- CTA 의도: Codex를 답변기가 아니라 실행 파트너로 이해하게 합니다.
- 화면 전달: 두 영역을 전체 비교로 보여준 뒤 Assistant에서 Agent로 이동합니다.
- Remotion 효과:
  - 0-5초: 비교 전체 유지
  - 5-12초: Assistant 영역 줌인
  - 12-22초: Agent 영역으로 팬
  - 22-27초: 전체로 복귀
- TTS 스크립트:
  Assistant와 Agent의 차이를 한 문장으로 정리해 보겠습니다. Assistant는 사용자가 질문하면 답을 제공합니다. 하지만 그 답을 복사하고 실행하고 저장하는 일은 여전히 사용자의 몫입니다. Agent는 한 단계 더 나아가 사용자가 해야 할 일을 대신 수행하고, 결과물까지 만들어 줍니다. Codex는 이 두 번째 방식, 즉 실행과 산출물에 가까운 도구입니다.

### 04. ChatGPT는 학습과 판단 기준을 빠르게 만들어줍니다

- 길이: 27초
- CTA 의도: 모르는 업무는 먼저 학습 가능한 기준으로 바꾸게 합니다.
- 화면 전달: 학습 화면에서 질문, 예시, 기준 정리 영역을 차례로 보여줍니다.
- Remotion 효과:
  - 0-5초: 학습 화면 전체에서 천천히 줌인
  - 5-11초: 질문 영역으로 팬
  - 11-19초: 중앙 예시 영역으로 이동
  - 19-27초: 정리 영역으로 팬 후 안정화
- TTS 스크립트:
  ChatGPT는 학습과 정리에 매우 강한 도구입니다. 예를 들어 엑셀의 브이룩업 함수가 낯설다면, 원리와 예시 데이터, 자주 하는 실수까지 한 번에 물어볼 수 있습니다. 이렇게 얻은 답변은 내가 더 잘 일하기 위한 기준이 됩니다. 즉 ChatGPT는 업무를 대신 끝내기보다는, 내가 업무를 이해하고 판단할 수 있도록 옆에서 도와주는 훌륭한 조력자입니다.

### 05. 답변만으로는 복사, 실행, 저장이 여전히 내 일입니다

- 길이: 28초
- CTA 의도: 반복 손작업을 Agent에게 넘길 필요를 느끼게 합니다.
- 화면 전달: 수작업 흐름 전체를 보여준 뒤 답변, 복사 실행, 저장 병목 순서로 이동합니다.
- Remotion 효과:
  - 0-5초: 수작업 흐름 전체 유지
  - 5-12초: 답변 영역 줌인
  - 12-20초: 중앙 작업 영역으로 팬
  - 20-28초: 결과 파일 쪽으로 이동
- TTS 스크립트:
  좋은 답변을 받았다고 해서 일이 끝나는 것은 아닙니다. 코드를 복사하고, 파일에 붙여 넣고, 실행하고, 저장하는 과정은 여전히 사람의 손에 남습니다. 한두 번이면 괜찮지만, 파일이 많아지고 반복이 늘어나면 이 손작업이 병목이 됩니다. 이 지점에서 우리는 에이전트가 왜 필요한지 이해하게 됩니다. 반복되는 실행 과정을 Codex에게 맡길 준비를 하는 것입니다.

### 06. ZIP을 풀고 HTML 가이드를 여는 것부터 시작합니다

- 길이: 26초
- CTA 의도: 모든 수강생이 같은 출발선에서 실습하게 합니다.
- 화면 전달: 세팅 전체에서 ZIP, HTML 가이드, 프로젝트 폴더 순서로 이동합니다.
- Remotion 효과:
  - 0-4초: 세팅 전체에서 시작
  - 4-10초: ZIP 압축 해제 단계로 줌인
  - 10-18초: HTML 가이드로 팬
  - 18-26초: 프로젝트 폴더 영역으로 이동
- TTS 스크립트:
  이제 Codex 실습 환경을 맞춰 보겠습니다. 강의에서는 모두가 같은 화면에서 시작하는 것이 중요합니다. 먼저 실습 키트 압축 파일을 풀고, HTML 가이드를 열어 전체 흐름을 확인합니다. 그다음 Codex에서 이 프로젝트 폴더를 지정합니다. 이렇게 해 두면 수강생마다 다른 파일 구조 때문에 생기는 혼란을 줄이고, 바로 실습에 집중할 수 있습니다.

### 07. 인앱 브라우저는 웹 내용을 읽고 행동 목록으로 바꿉니다

- 길이: 26초
- CTA 의도: 웹페이지를 단순 요약이 아니라 다음 행동으로 바꾸게 합니다.
- 화면 전달: 브라우저 전체에서 페이지 열기, 핵심 추출, 행동 목록 순서로 팬합니다.
- Remotion 효과:
  - 0-4초: 브라우저 전체 유지
  - 4-10초: 웹페이지 영역으로 줌인
  - 10-18초: 중앙 정보 영역으로 팬
  - 18-26초: 오른쪽 산출물로 이동
- TTS 스크립트:
  첫 번째 실습은 인앱 브라우저입니다. 브라우저는 단순히 웹페이지를 보는 공간이 아닙니다. Codex에게 페이지를 열고, 핵심 내용을 추출하고, 다음 행동 목록으로 정리하라고 요청할 수 있습니다. 병원 공지, 자료실, 업무 지침 페이지처럼 정보가 흩어져 있는 곳에서 특히 유용합니다. 보는 일을 행동 가능한 브리핑으로 바꾸는 연습입니다.

### 08. 컴퓨터 정리는 제안, 검토, 실행 순서로 갑니다

- 길이: 25초
- CTA 의도: 위험한 자동화일수록 검토 가능한 절차가 먼저라는 기준을 줍니다.
- 화면 전달: 정리 전체에서 후보 파일, 안전 검토, 승인 실행 순서로 이동합니다.
- Remotion 효과:
  - 0-4초: 정리 전체에서 시작
  - 4-10초: 파일 분류 영역으로 팬
  - 10-18초: 보안 검토 영역 줌인
  - 18-25초: 결과 체크로 이동
- TTS 스크립트:
  두 번째 실습은 컴퓨터 정리입니다. 여기서 중요한 원칙은 바로 실행하지 않는 것입니다. 파일 삭제나 이동은 위험할 수 있으니, 먼저 정리 기준을 만들고, 드라이 런으로 후보 목록을 확인합니다. 그다음 사람이 승인한 뒤 실행합니다. 자동화의 핵심은 빠르게 지우는 것이 아니라, 검토 가능한 안전한 절차를 만드는 것입니다.

### 09. 근무표 이미지는 구조화 표와 Calendar 후보로 바꿉니다

- 길이: 28초
- CTA 의도: 근무표 이미지를 실제 일정 데이터로 바꾸는 과정을 이해시킵니다.
- 화면 전달: 원본 근무표 전체를 보여준 뒤 날짜와 근무 코드 행을 확대하고, Calendar 후보로 이어지는 흐름을 설명합니다.
- Remotion 효과:
  - 0-5초: 원본 근무표 전체 contain 유지
  - 5-13초: 근무 코드 행으로 강하게 줌인
  - 13-21초: 표의 핵심 구간을 따라 팬
  - 21-28초: 전체로 줌아웃
- TTS 스크립트:
  세 번째 실습은 근무표 이미지입니다. 이미지는 사람이 눈으로 읽을 수 있지만, 캘린더 앱은 바로 이해하지 못합니다. 그래서 Codex에게 먼저 근무표 이미지를 확인하게 하고, 날짜와 근무 코드를 구조화된 표로 추출하도록 요청합니다. 마지막으로 구글 캘린더에 넣을 수 있는 일정 후보로 정리합니다. 이미지 속 정보를 실제 일정 데이터로 바꾸는 과정입니다.

### 10. AI 일자리 데이터는 Excel 분석과 차트로 설득합니다

- 길이: 27초
- CTA 의도: 숫자 데이터를 보고와 강의에 쓸 수 있는 메시지로 바꾸게 합니다.
- 화면 전달: 원천 데이터에서 Excel 정리 화면을 거쳐 차트와 인사이트 영역으로 이동합니다.
- Remotion 효과:
  - 0-4초: 분석 전체에서 시작
  - 4-10초: 왼쪽 자료 영역으로 팬
  - 10-18초: 노트북 표로 줌인
  - 18-27초: 오른쪽 차트 영역으로 이동
- TTS 스크립트:
  네 번째 실습은 데이터 분석입니다. 표 데이터는 그 자체로는 설득력이 약할 수 있습니다. 공개 자료를 CSV로 확인하고, Excel에서 정리한 다음, 차트와 인사이트로 바꾸어야 보고나 강의에 사용할 수 있습니다. Codex에게는 단순 요약이 아니라, 분석 파일을 만들고 차트까지 포함하라고 요청할 수 있습니다. 숫자를 메시지로 바꾸는 연습입니다.

### 11. 환자 안내문은 3문장과 3장 카드뉴스로 바꿉니다

- 길이: 28초
- CTA 의도: 긴 의료 안내문을 환자가 바로 이해하는 카드뉴스로 재구성하게 합니다.
- 화면 전달: 카드뉴스 3장을 전체로 보여준 뒤 1장, 2장, 3장을 순서대로 훑습니다.
- Remotion 효과:
  - 0-5초: 세 카드 전체 등장
  - 5-12초: 첫 카드로 줌인
  - 12-20초: 두 번째 카드로 팬
  - 20-28초: 세 번째 카드 후 전체 복귀
- TTS 스크립트:
  다섯 번째 실습은 환자 안내문입니다. 긴 안내문은 정확하지만, 환자가 바로 행동하기에는 부담스러울 수 있습니다. Codex에게 문서를 읽고 핵심을 세 문장으로 줄인 뒤, 카드뉴스 세 장으로 나누게 합니다. 이때 의료 정보는 과장하지 않고, 환자가 무엇을 준비하고 어떤 점을 주의해야 하는지에 집중합니다. 긴 문서를 이해하기 쉬운 시각 자료로 바꾸는 과정입니다.

### 13. Codex에게 맡길 때는 다섯 칸만 채우면 됩니다

- 길이: 26초
- CTA 의도: 긴 프롬프트보다 빠진 칸 없는 프롬프트를 만들게 합니다.

  - 0-4초: 전체 공식 표시
  - 4-9초: 첫 칸으로 줌인
  - 9-15초: 작업과 산출물 칸으로 팬
  - 15-22초: 제약과 검토 칸으로 이동
